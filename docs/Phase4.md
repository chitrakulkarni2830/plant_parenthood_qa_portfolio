# Phase 4: Core Features

## Objective
Implement and test advanced data manipulation features to support complex testing scenarios.

## Scope
* Search, Filters, Categories, Plant Profiles, Watering Schedules, Harvest Tracking.

## Features
* Search bar functionality.
* Filter by category.
* Detailed plant profile views.

## Learning Topics
* Advanced JS array filtering vs SQL `WHERE` clauses.
* URL Query Parameters.

## Frontend Work
* Build UI for search, filters, and individual plant views.

## Backend Work
* API support for query parameters (e.g., `/api/plants?category=herbs`).

## Testing Activities
* End-to-end manual testing of search and filter logic.
* JIRA bug reporting for boundary values (e.g., searching special characters).

## API Activities
* Parameterized GET requests.

## SQL Activities
* `LIKE` operators and complex `JOIN`s for plant profiles and watering logs.

## Playwright Activities
* *Future prep:* Draft test scenarios for UI search and filtering.

## Deliverables & GitHub Artifacts
* Feature branches for Search, Filters, and Profiles.
* Updated test cases for new features.

## Completion Criteria
* Users can accurately search and filter the 60-70 plant database from the UI.
