# Phase 5: UI Polish & Manual Regression

## Objective
Finalize the UI and conduct a full manual regression test across the entire application before introducing automation.

## Scope
* CSS styling and layout fixes.
* Comprehensive manual regression suite execution.

## Features
* Responsive UI (Mobile/Desktop views).

## Learning Topics
* CSS Grid/Flexbox.
* QA Regression cycles.

## Frontend Work
* Styling application. Adding semantic HTML (`data-testid` attributes) to prepare for Playwright.

## Backend Work
* Bug fixes based on testing findings.

## Testing Activities
* Execute all test cases from Phases 1-4.
* JIRA defect management (verify fixes, transition tickets).

## API Activities
* Verify no UI changes broke API payloads.

## SQL Activities
* Verify DB constraints hold up against edge-case UI inputs.

## Playwright Activities
* **Final check:** Ensure all buttons, inputs, and elements have robust selectors (IDs, data-attributes).

## Deliverables & GitHub Artifacts
* Polished CSS files.
* Finalized Manual Regression Test Report.

## Completion Criteria
* Zero critical or high severity defects remain open in JIRA.
* UI is stable and ready for automation.
